# Minimum Collaborator Permissions

Request only the permissions needed to activate and maintain the headless storefront.

## Request

- **Products** — view, create, edit, publish to sales channels
- **Collections** — view, create, edit
- **Files** — view and upload product or brand assets if needed
- **Content** — review pages, menus, and policy content for launch alignment
- **Metafields and metaobjects** — create/update custom scent metadata
- **Apps and sales channels** — install/approve Headless channel and review app compatibility
- **Online Store** — review current theme and checkout behavior for rollback reference
- **Store settings** — domain, checkout, and sales channel settings needed for activation review

## Do not request

- Payout access
- Billing access
- Unnecessary customer data access
- Unnecessary order access beyond checkout testing approved by the owner

## Why these permissions are enough

The Next.js storefront uses the Storefront API for catalog, cart, and checkout. Admin API access is optional and only needed for the metafield sync script or webhook administration.
